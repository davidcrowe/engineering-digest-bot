# Engineering Digest Bot

Reads daily Slack #engineering messages and posts a structured end-of-day summary with decisions, questions, blockers, and top contributors.

> Generated by [MakeAgents](https://makeagents.run). MIT-licensed — this code is yours.

## What you get from ACP

Every tool call is wrapped with [`@agenticcontrolplane/governance-anthropic`](https://www.npmjs.com/package/@agenticcontrolplane/governance-anthropic):

- **Deny / redact before execution** — denied calls never run. Claude sees `tool_error`.
- **Per-user policy** — the JWT you pass to `/run` is the actor on every audit record.
- **Rate limits and budgets** — enforced server-side, per user.
- **PII detection** — tool inputs (pre) and outputs (post) are scanned.
- **Unified audit log** — visible at [`/activity`](https://cloud.agenticcontrolplane.com/activity).

Outbound calls to vendor APIs use [`@agenticcontrolplane/proxy`](https://www.npmjs.com/package/@agenticcontrolplane/proxy) — your agent holds only opaque short-lived tokens; real OAuth credentials never leave the gateway.

## Run it

```bash
npm install
cp .env.example .env   # fill in ANTHROPIC_API_KEY
npm run dev
```

## Call it

```bash
curl -X POST http://localhost:8000/run \
  -H "Authorization: Bearer <user-jwt>" \
  -H "Content-Type: application/json" \
  -d '{"prompt": "your prompt here"}'
```

The Bearer token must be a JWT issued by your tenant's configured IdP.

## Configuration

- **Model:** `gemini-2.5-pro`
- **Tools:** `slack.getChannelHistory`, `slack.searchMessages`, `slack.sendMessage`, `memory.save`, `memory.search`

## Going ACP-free

Want to run without ACP? You'll lose governance, audit, and credential brokering — but the code is yours, here's how:

1. Replace `governHandlers({...})` with plain handlers (drop the wrapper).
2. Remove the `withContext({ userToken: token }, ...)` wrap around the loop.
3. Drop `@agenticcontrolplane/governance-anthropic` and `@agenticcontrolplane/proxy` from `package.json`.
4. For tools that called external APIs via `acpFetch`, replace them with direct `fetch` calls + your own OAuth tokens (set per-vendor env vars).
